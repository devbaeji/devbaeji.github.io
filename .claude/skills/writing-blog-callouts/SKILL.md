# 블로그 callout 작성 규칙

devbaeji.github.io (Jekyll + Chirpy 테마) 블로그의 callout(prompt 박스) 작성 표준이다.
모든 `_posts/*.md` 파일에서 callout을 추가/편집할 때 이 규칙을 따른다.

## 공통 컴포넌트

Chirpy 테마가 내장한 `prompt` 컴포넌트를 표준 callout으로 사용한다.
**별도 CSS는 만들지 않는다.** Markdown 끝에 `{: .prompt-* }` 마커 한 줄로 스타일이 적용된다.

| 마커 | 색상 | 용도 |
|---|---|---|
| `.prompt-info` | 파랑 | 정의 박스, Q&A 박스의 기본 |
| `.prompt-tip` | 초록 | 의사결정 가이드, 인사이트 |
| `.prompt-warning` | 노랑 | 운영 시 주의사항 |
| `.prompt-danger` | 빨강 | 흔한 함정, 오해 |

## 두 가지 callout 형식

### 1. 정의 박스 — 개념/용어를 박스로 정리

본문에서 새로 등장한 개념을 박스로 부연한다. 박스 첫 줄에 **개념명을 굵게** 둔다.

```markdown
> **Cosine Similarity**
>
> 두 벡터가 가리키는 방향이 얼마나 일치하는지 측정한다.
> 값의 범위는 -1 ~ 1. 1에 가까우면 같은 방향, 0이면 직교(무관), -1이면 정반대.
>
> 벡터의 길이를 무시하고 방향만 보기 때문에 텍스트 임베딩에서 표준으로 쓰인다.
{: .prompt-info }
```

### 2. Q&A 박스 — 실무에서 자주 마주칠 질문 + 모범 답안

블로그를 읽고 자연스럽게 떠오를 만한 질문에, 결론 → 흐름/근거 → 측면 효과 순으로 **완결된 답을 정리한다.** 질문만 던지고 끝내지 않는다.
면접/회의/온콜에서 실제로 답할 수 있는 수준이어야 한다.

```markdown
> **Q.** 화이트리스트가 정적 CSS 파일에서 추출된다. 패키지가 새 버전으로 install되면 ESLint가 자동으로 새 토큰을 알 수 있나?
>
> 알게 된다.
>
> 흐름은 단순하다. `pnpm install @org/ui-package@latest`로 CSS가 새 버전으로 교체되면, ESLint를 다시 실행할 때 `readFileSync`가 새 CSS를 읽고 화이트리스트가 자동 갱신된다.
>
> 단, 에디터에서 실시간 반영되려면 ESLint 서버 재시작이 필요하다. IntelliJ는 "Restart ESLint" 명령을 수동으로 눌러야 한다.
>
> *측면 효과가 진짜 가치다.* 새 토큰 추가가 코드 push만으로 완결된다. 룰 코드는 한 줄도 안 바뀐다.
{: .prompt-info }
```

## Q&A 박스의 4단 구조 (필수)

1. **`Q.` 구체적·실무적 질문 한 줄** — 모호한 질문 금지. "어떻게 잘 하나?"가 아니라 "X와 Y 중 무엇을 고르나?"
2. **결론 한 줄** — 단정적으로. "알게 된다", "실제로 그렇다", "두 가지 방식이 있다", "못한다"
3. **흐름/근거** — 메커니즘, 코드, 단계. 필요하면 "단, ..."으로 예외/조건 추가
4. **측면 효과 / 진짜 가치** — `*기울임*`으로 강조 (예: `*측면 효과가 진짜 가치다.*`). 답변의 차별점이 되는 인사이트

## 배치 원칙

- **본문 흐름 중간**에 배치한다. 글 끝에 모아두지 않는다.
- 정의 박스 → 해당 개념이 본문에 처음 등장한 직후
- Q&A 박스 → 본문이 다룬 주제 직후, 자연스럽게 따라오는 의문 지점
- 한 글에 **3~6개**가 적절하다. 정의 박스와 Q&A 박스를 섞는다.

## 금지 표현

- 박스 머릿글에 분류 라벨 금지: ❌ "용어 정리", "심화 토픽", "면접 예상 질문", "필독", "체크리스트"
- 학습 가이드 톤 금지: ❌ "이 글을 이해했다면 다음에 답할 수 있어야 한다"
- 박스 머릿글은 오직 두 가지뿐:
  - 정의 박스 → `**개념명**`
  - Q&A 박스 → `**Q.** 질문`

## 체크리스트

callout을 추가/편집한 뒤 다음을 확인한다.

- [ ] 박스 머릿글이 분류 라벨이 아니라 개념명 또는 `Q.`로 시작하는가?
- [ ] Q&A 박스라면 결론 한 줄 → 흐름/근거 → 측면 효과 4단 구조를 모두 채웠는가?
- [ ] 측면 효과 부분이 `*기울임*`으로 강조되어 있는가?
- [ ] callout이 본문 흐름 중간에 자연스럽게 배치됐는가? 글 끝에 모여 있지는 않은가?
- [ ] `{: .prompt-* }` 마커가 박스 끝에 붙어 있는가?
- [ ] 한 글에 callout이 3~6개 범위인가?

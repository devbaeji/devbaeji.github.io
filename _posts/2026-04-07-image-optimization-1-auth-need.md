## 문제

100×100 썸네일이 3MB 원본을 그대로 내려받고 있었다.

## 제약

- **파일별 권한 모델** — 워크스페이스/티켓 단위. 같은 회사여도 열람 불가한 파일이 있음
- **Private S3** — 백엔드가 권한 검증 후 스트리밍. 인증/인가 로직이 여기에 집중
- **트래픽 규모** — CDN 도입 ROI가 안 나옴

## 선택지

| 방식 | 장점 | 문제 |
|---|---|---|
| CDN + Signed URL | CDN 캐싱/변환 | 만료 관리, 권한 변경 반영 지연, URL 노출, 추가 비용 |
| 매 요청 Presigned URL | 단순 | 노출/만료 이슈 동일, 변환 별도 필요 |
| **쿠키 + 백엔드 프록시** | 기존 인증 재사용, 권한 즉시 반영 | 리사이즈 필수 |

## 결정: 쿠키 + 백엔드 프록시

```
브라우저 (쿠키) → /api/files/stream/{id} → 권한 체크 → S3 → 응답
```

CDN은 "URL을 알면 누구나 접근 가능"이라는 전제 위에 있다. 그 전제가 깨지는 서비스에서 CDN을 쓰려면 인증 모델을 재설계해야 한다. 재설계 비용 > 현재 트래픽 비용이면, 기존 모델 재사용이 맞다.

대신 두 가지 서브 과제가 붙는다.

1. 클라이언트가 **필요한 크기만** 요청
2. 같은 크기 요청은 **캐시 재사용**

## 핵심

> 인증 모델이 이미지 아키텍처를 결정한다.
